The CSV is loaded onto the PVC at runtime, not baked into the image. If the pods
are stuck in Init (waiting for {{ .Values.csv.fileName }}), load the data:
  make load-streamer-data

The streamer is a Kafka producer — it exposes no Service, only health probes on
port {{ .Values.healthPort }} (/healthz, /readyz, /stats).

Watch it scale and stream:
  kubectl -n {{ .Release.Namespace }} get hpa,pods -l app.kubernetes.io/name={{ include "app.name" . }} -w

Inspect throughput counters on a pod:
  kubectl -n {{ .Release.Namespace }} port-forward deploy/{{ include "app.fullname" . }} {{ .Values.healthPort }}:{{ .Values.healthPort }}
  curl localhost:{{ .Values.healthPort }}/stats

NOTE: each replica streams the full dataset independently, so the fleet's
aggregate rate is (replicas / {{ .Values.streamer.interval }}). Scaling Streamers
raises the produced load; scale Collectors to keep up. Provision the topic with
>= maxReplicas ({{ .Values.autoscaling.maxReplicas }}) partitions so collector
parallelism is not the bottleneck.
